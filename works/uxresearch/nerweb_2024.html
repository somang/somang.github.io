2024, Designing user interface for caption quality evaluation tool

The Canadian Radio-television and Telecommunications Commission (CRTC) regulates the quality of Closed Captioning (link), which uses a specific model to assess. The assessment requires certified evaluators to compare the televised captions with the transcript, and generate a quality report for the broadcasters to audit the quality. In this project, a software tool was designed using the User Centre Design principles. It iterated two cycles of user research, including the System Usability Scale (SUS) and user studies with initial user interface design and an Alpha prototype. Total duration: 11 months

1. Requirement gathering

From the initial stage, three objectives were focused to develop primitive interface.

  1. Task requirements
  2. Desired workflow
  3. Need, Want, and Wish lists
Starting from the baseline knowledge, the fact that the Canadian NER model process was studied to learn the desired workflow. Then two experts were contacted for the initial interview. The experts did not want to participate in the actual interview process, however, gave a list of painpoints that they want to solve from the current tool and process. In addition, it was possible to acquire the existing tool that users were using (e.g., an Excel VBA script). With these, an initial task requirement lists were created.

2. Prototyping with the baseline knowledge

Initially, the Canadian NER model quality assessment procedure was studied to see how error severity was determined and how each error types were defined. The requirement lists and benchmarking of some other existing tools for other purposes were taken as design references. This allowed us to go through initial low fidelity prototype and potential user journey.

High fidelity prototype

Due to time and resource limitation, the initial study was only done in 6 month in for the project. The project had to work with the software development team in parallel, and left no time to be wasted. Therefore, the sketches were evaluated within the team, and the initial proof of concept level of software was implemented to be tested with the expert users. Then, the first high fidelity prototype was created.

In order to communicate with the stakeholders, a user journey of the desired system was also presented.

3. First user study

The first user study was planned to test the usability of the interface and system, as well as to learn deeper into the experts workflow and desire. Therefore, the study was designed as the following order:

  1. Demographic questionnaire
  2. System Usability Scale (SUS) on the current interface
  3. User tasks (think aloud protocol) with semi-structured interview questions
User tasks were split to three high-level topics. 1) Sign up & sign in, 2) Creating project, and 3) Performing the evaluation using the system. At each page transition, participants were aksed to describe their expected system behaviour, as well as what they would do with the presented interface. The qualitative data was collected using video and audio, then analyzed using Condens for thematic open coding.

4. Design recommendations and after...

From the first round of user study, a set of design recommendations were found. These design recommendations were then presented and discussed with the software development team, followed by interface update on the tool. Upon implementation, a second round of user study was performed with the same five expert users. The second study was designed as the following order:

  1. User tasks (think aloud protocol) with semi-structured interview questions
  2. System Usability Scale (SUS) on the NERWEB interface
The final version specifically asked about the user experience in seeing AI generated quality assessment to be used as decision support tool in quality assessment. The user reaction towards AI system was positive, which was also highlighted in their reaction in how it was presented at the end of their own evaluation.

Finally

The SUS score comparison showed that there was an improved usability from the conventional interface to the NERWEB interface, which was already proven from participants reaction in the interview.
This work was completed and all user research process is described in detail for academic publication manuscripts.
